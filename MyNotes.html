  1. Introduction

    1. The World Wide Web

      The world wide web can be modeled using three major pieces: There is you and your browser, there is the internet and there are servers. Whenever you want to access a webpage your browser makes requests to a server via the internet using a protocol called http. The server responds with files that the browser displays on your computer. Those files are coded in HTML (Hyper Text Markup Language) among other languages. Hyperlinks allow you to move from one webpage to another, thus making the whole thing weblike. The number of webpages existing is somewhere on the order of 30 billion.

    2. Web Pages

      On this page we are going to focus on how web pages are coded. One can basically think of a web page as a house, then there is
      • HTML, which describes the structure of the house, like "Where are the walls?" or "Which room is the living room?",
      • CSS, which describes the style of the house, like "Are there any curtains?" or "Which colour is that wall?",
      • Java Script, which describes the interactive elements, like "Open a door!",
      • rules, like "A TV cannot float in a room." and
      • tools, like "You need a hammer to build a wall."
      This text is going to give an introduction to HTML and CSS as well as some brief thoughts on how to learn a programming language.

  2. HTML - Hypertext Markup Language

    HTML files define the structure of a web page. This structure is tree like, meaning it is a structure that has branches. The DOM - Document Object Model is the convention for the browser for representing and interacting with elements in HTML. An element is the most basic structure in HTML and is represented through tags. Elements can be empty, contain text or contain other elements. Each tag has a type, which defines what kind of element will be created by the browser, e.g. a paragraph or an image. Tags can also have attributes describing them. In order to give your web page styles you will use a different language called CSS, which will be described later in the text.

    1. HTML - Tags and Attributes

      An HTML element basically consists of tags and contents:
      < NAME > Contents < /NAME>
      The first red part is called an opening tag, the second one a closing tag. The contents are inbetween the tags. However, there are tags that do not contain any contents, these are called void tags. Void tags do not have closing tags either. Some examples are
      • <b> Contents </b> - Contents will be bold
      • <em> Contents </em> - Contents will be in italics (emphasized)
      • <br> - creates a new line (This is an example of a void tag)
      • <em> Contents </em> - Contents will be contained in one paragraph

      Some tags have attributes:
      < TAG ATTR=" value" > Contents < /TAG>
      Examples include
      • <a href="www.reddit.com"> Contents </a> - the anchor-tag, which is used for hyperlinks
      • <img src="url" alt="text"> Contents </img> - the image-tag, which is used to for images, "text" is displayed in case of broken requests and is also useful for blind people

    2. Inline vs Block Tags

      Tags like <b> , <em> , <img> and <br> are called inline, which basically means that the contents if present stay within the same line. <p> is an example of a block tag. Block tags create some sort of an invisible box around the contents, there will be more detail on that later.

    3. Container Elements

      Container elements are elements that do nothing but contain stuff inside them. Two of them are <span>text</span>, which is inline and <div>text</div>, which is a block tag.

    4. Boxes and Grids

      The background of a web page is basicly a grid in which the elements are arranged as boxes, that can have sub-boxes and so on. So the first box is the web page itself. The <div>text</div> tag defines a box. As later you will want to add style to your boxes you have to label the boxes. That is where the class- attribute comes in, e.g. <div class="something">text</div>. Boxes can have several labels as they can belong to multiple classes.

    6. Miscellania

      Whitespaces

      In HTML all whitespaces or sequences of whitespaces turn into a single space.

      Escape Codes

      Sometimes in a text you will want to write something like "< b >" instead of of having those characters be interpreted as HTML code. That is where escape codes come in. This example would be written as "&lt; b &gt;".

  3. CSS - Cascading Style Sheets

    1. Introduction

      Cascading Style Sheets let you add style to your web page, that is they let you control how your web page is going to look through socalled rules. Cascading means that you go down and down with all the rules for an element until you find the most specific rule which then gets applied.

    3. The Box Revisited

      As mentioned above everything on a web page is a box.
      A box consists of a margin, a border, a padding and the content, each one containing the other. The content is the image or the text you want to be displayed. The padding clears out an area around the content and is affected by the background color of the box. The border surrounds the padding and the margin clears up an area around the border, is essentially trasparent and defines the space between the boxes.
      By default the size of a box is calculated as "border+padding+content", but there is an easier to use model available, which allows you to enter the size of the box that then stays fixed when tweaking the border and the padding. You can use this model by setting box-sizing:border-box in your CSS-file.

    4. Positioning Boxes

      As a block element a div-box will by default use up the whole of the width of the page. Adding the style display:flex will override this behaviour and let you put several boxes on one line.

  4. Developer Tools

    Browsers such as Firefox or Chrome offer socalled developer tools to visualize the structure of a web page.
    Sideways triangles symbolize elements and can be expanded to show the contents. Hovering over an element with the mouse leads to the according part of the page being highlighted. You can also use those tools to display the styles of a page.

  5. Philosophy for Programmers

    1. Computers are Stupid

      In spoken languages even grammatically correct sentences can contain some sort of ambiguity. Humans usually infer the correct meaning by looking at the context. In contrast to humans computers interpret code literally.

    2. It's OK to Forget

      In programming the key is to understand the big ideas. It does not matter if you do not know each and every command, instead it is important to understand the concepts and to know where to find the things you need.

    3. Structuring Code

      In order to keep your code readable it is important to give it a certain structure, this is where indenting comes in. Remember, it is not only the machine that has to "understand" your code, a colleage or maybe even yourself half a year later might want to read your code too.

    4. Don't Repeat Yourself

      The idea is to reuse code as you go along programming. I.e. you can use the same style several times while only defining it once.

    5. Some Thoughts on Learning

      One can model a learning process using four steps
      • Ignorance
      • Awareness
      • Ability
      • Fluency
      While getting from ignorance to awareness and to ability takes only a rather short time it can take ages to actually achieve fluency.
